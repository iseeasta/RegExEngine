--> PATTERN 2 :  a+b+
    Given Output  :  "aaaabb"
    Expected      :  one or more 'a' then one or more 'b'. like "ab" or "aaabb"
    Result        :  PASS

    Both (+) cases fired correctly. <---\n--->
    First (+) grabbed 'a' and repeated it. Second (+) grabbed 'b' and repeated it. <---\n--->
    But wait, there is a hidden problem here. <---\n--->
    The (+) case does (sb.append) meaning it ADDS on top of whatever is already in sb. <---\n--->
    But (patternCh[i-1]) gave us 'a' or 'b' which were already appended by the literal else branch. <---\n--->
    So the output is actually DOUBLING the last literal. "a" gets appended once by else, <---\n--->
    then (+) appends "aaa" more on top. Result is fine by coincidence here but the logic is shaky.

--> PATTERN 3 :  a?b?c
    Given Output  :  "aabc"
    Expected      :  optional 'a', optional 'b', then 'c'. like "c" or "bc" or "abc"
    Result        :  FAIL

    Problem: output is "aabc" but 'a' should appear 0 or 1 time max, not 2 times. <---\n--->
    Same doubling issue as pattern 2. <---\n--->
    The literal else branch appends 'a' first. then (?) case appends 'a' again (1 time). <---\n--->
    So 'a' ends up appearing twice even though (?) means max 1. <---\n--->
    Root cause: (+), (*), (?) cases should NOT re-append the prev char. <---\n--->
    The prev char was already appended by the else branch when it was processed as a literal. <---\n--->
    Fix needed: before appending in (+)(*)(?) cases, DELETE the last appended char from sb first. <---\n--->
    like: (sb.deleteCharAt(sb.length() - 1)) before doing the repeat append.

--> PATTERN 4 :  [abc]+
    Given Output  :  "c]"
    Expected      :  one or more chars from set a,b,c. like "a" or "cab" or "bbc"
    Result        :  FAIL

    TWO problems here: <---\n--->

    Problem 1 - the ']' is appearing in output. <---\n--->
    After the [ case finishes, (i) is sitting ON the ']' character. <---\n--->
    Then the for loop does (i++) moving past ']'. <---\n--->
    But then (+) fires and looks at (patternCh[i-1]) which is now ']'. <---\n--->
    So pChar = ']' and it gets appended. <---\n--->
    Fix needed: [ case should leave (i) ON the ']' so the for loop's (i++) lands past it cleanly, <---\n--->
    which it already does. The real issue is (+) blindly taking patternCh[i-1] = ']'. <---\n--->
    (+) needs to know the ACTUAL preceding token, not just the char before it in the raw pattern.

    Problem 2 - [ case only picks 1 char but (+) should make it repeat 1 or more times. <---\n--->
    Currently [ picks 1 char, then (+) fires and repeats ']' instead of repeating from the set. <---\n--->
    Fix needed: [ case needs to detect if next char after ] is (+)(*)(?) and handle repetition itself, <---\n--->
    same way ( case already peeks at next char and handles its own quantifier.

    Best fix is: [ case handles its own {} quantifier by peeking ahead, same as ( case does.
